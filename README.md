# الشركة المتحدة لقطع غيار التبريد والتكييف والغسالات وتوريدات عامة

موقع ويب ثابت (Static — HTML/CSS/JavaScript فقط، بدون سيرفر أو قاعدة بيانات)، مُعَدّ ليتم نشره مباشرة عبر **GitHub Pages**.

## المحتوى

- عرض قطع الغيار مصنّفة حسب: قطع غيار تكييفات، قطع غيار تبريد، قطع غيار غسالات، وتوريدات عامة.
- بحث فوري وفلاتر (التصنيف، التصنيف الفرعي، الماركة، الموديل).
- سلة اختيارات لتجميع عدة قطع، ثم إرسال طلب كشف سعر منظم عبر واتساب مباشرة.
- زر "طلب سريع واتساب" لكل قطعة بشكل منفرد.
- صفحة مخصصة لخدمات التوريد للشركات والمصانع.
- سكربت بايثون لتحديث المنتجات تلقائياً من ملف Excel/CSV (راجع `tools/README.md`).

## تشغيل الموقع محلياً

بما أن الموقع Static بالكامل، تكفي أي أداة تشغّل سيرفر محلي بسيط لأن المتصفح يمنع قراءة ملفات JSON مباشرة عبر `file://`. من داخل مجلد المشروع:

```bash
python3 -m http.server 8000
```

ثم افتح المتصفح على:

```
http://localhost:8000
```

## رفع الموقع على GitHub Pages

1. أنشئ مستودع (Repository) جديد على GitHub.
2. ارفع كل ملفات المشروع لهذا المستودع:
   ```bash
   git init
   git add .
   git commit -m "نشر الموقع الأول"
   git branch -M main
   git remote add origin https://github.com/USERNAME/REPO-NAME.git
   git push -u origin main
   ```
3. من إعدادات المستودع على GitHub: **Settings → Pages**، اختر Branch: `main` والمجلد `/ (root)`، ثم احفظ.
4. سيظهر رابط الموقع المباشر خلال دقائق بصيغة:
   ```
   https://USERNAME.github.io/REPO-NAME/
   ```

## لوحة تحكم الأدمن (إضافة منتجات وتصنيفات وصور)

تم إضافة لوحة تحكم بسيطة تعمل بالكامل من المتصفح (بدون سيرفر):

- رابط الدخول: `admin/login.html` (وهناك رابط خافت باسم "لوحة التحكم" في فوتر كل صفحة).
- بيانات الدخول الافتراضية: **hos** / **111**.
- من تبويب **الإعدادات** داخل اللوحة يمكن تغيير اسم المستخدم وكلمة المرور في أي وقت.
- من تبويب **المنتجات** و **التصنيفات** يمكن إضافة/تعديل/حذف المنتجات والتصنيفات ورفع صورة لكل منتج مباشرة من الجهاز.

### ⚠️ نقطة مهمة جداً (لأن الموقع Static بدون قاعدة بيانات)

كل تعديل يتم من اللوحة يُحفظ داخل متصفحك فقط (localStorage)، وتراه فوراً في نفس المتصفح على الموقع كمعاينة حية. لكنه **لن يظهر تلقائياً لباقي الزوار** لأن الموقع مستضاف كملفات ثابتة (GitHub Pages).

لجعل أي تعديل ظاهراً بشكل دائم لكل الزوار:
1. ادخل تبويب **"تصدير / استيراد"** داخل اللوحة.
2. اضغط تنزيل `products.json` و/أو `categories.json`.
3. استبدل بهما نفس الملفين داخل مجلد `data/` في مشروعك.
4. ارفع التعديل (Commit + Push) إلى GitHub ليظهر على الرابط العام للجميع.

هذا القيد موجود لأن الموقع لا يحتوي على سيرفر أو قاعدة بيانات؛ لو أردت لاحقاً أن تُحفظ التعديلات تلقائياً للجميع بدون تصدير يدوي، سيحتاج ذلك لإضافة قاعدة بيانات/سيرفر خلفي (مثل Firebase أو باك إند مخصص)، وهو أمر يمكن تنفيذه في مرحلة لاحقة إذا رغبت.

## ملاحظات مهمة

- **رقم الواتساب** وبيانات الشركة معرّفة في مكان واحد بأعلى ملف `assets/js/render.js` (المتغيّر `SITE`) لسهولة التعديل مستقبلاً.
- **روابط التواصل الاجتماعي** في الفوتر بقيمة `#` مؤقتاً — ابحث عن التعليق `TODO: استبدل الروابط هنا` في ملفات HTML لاستبدالها بحسابات حقيقية.
- **الصور**: أي قطعة بلا صورة حقيقية تعرض تلقائياً صورة placeholder عامة (`assets/images/products/placeholder.svg`) حتى يتم رفع الصورة الحقيقية.
