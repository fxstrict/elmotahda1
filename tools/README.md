# كيفية إضافة أو تحديث قطع الغيار

هذا الشرح موجّه لصاحب الشركة أو أي موظف يريد إضافة قطع غيار جديدة للموقع **بدون أي تعديل في الأكواد البرمجية**.

## الخطوات

### 1. جهّز ملف Excel أو CSV

أنشئ ملف Excel باسم مثلاً `products.xlsx` (أو CSV)، وضع فيه الأعمدة التالية بالضبط في الصف الأول (بالإنجليزية كما هي):

| code | name | model | brand | category | subcategory | image_filename | description |
|------|------|-------|-------|----------|-------------|-----------------|--------------|

**شرح الأعمدة:**

- **code**: كود القطعة (مثال: `AC-COMP-050`) — يجب أن يكون فريداً لكل قطعة.
- **name**: اسم القطعة بالعربية (مثال: `كباس تكييف 2 حصان`).
- **model**: الموديلات المتوافقة (مثال: `متوافق مع: LG / Samsung`).
- **brand**: اسم الماركة.
- **category**: معرّف التصنيف الرئيسي، ويجب أن يطابق أحد المعرّفات التالية بالضبط:
  - `ac-parts` (قطع غيار تكييفات)
  - `refrigeration-parts` (قطع غيار تبريد)
  - `washing-machine-parts` (قطع غيار غسالات)
  - `general-supplies` (توريدات عامة)
- **subcategory**: التصنيف الفرعي بالعربية (مثال: `كباسات (كمبروسورات)`).
- **image_filename**: اسم ملف الصورة فقط (مثال: `AC-COMP-050.jpg`) — اتركه فارغاً وسيستخدم النظام تلقائياً `code.jpg`.
- **description**: وصف مختصر اختياري للقطعة.

### 2. ارفع صور القطع الحقيقية (اختياري لكن يُفضّل)

ضع صور القطع في المجلد:

```
assets/images/products/
```

بحيث يكون اسم كل صورة **مطابقاً لكود القطعة** بالضبط، مثال: `AC-COMP-050.jpg`.

إذا لم تتوفر صورة حقيقية لقطعة معينة، لا مشكلة — سيظهر للزائر صورة عامة (placeholder) تلقائياً حتى يتم رفع الصورة الحقيقية لاحقاً.

### 3. شغّل أمر واحد فقط

من داخل مجلد المشروع الرئيسي، شغّل:

```bash
python3 tools/excel_to_json.py products.xlsx
```

أو إذا كان الملف CSV:

```bash
python3 tools/excel_to_json.py products.csv
```

> ملاحظة: إذا ظهرت رسالة تطلب تثبيت مكتبة `openpyxl`، شغّل أولاً:
> ```bash
> pip install openpyxl --break-system-packages
> ```

سيقوم السكربت تلقائياً بـ:
- قراءة كل صف وتحويله لقطعة غيار جديدة في `data/products.json`.
- تحديث أي قطعة موجودة بالفعل بنفس الكود (بدلاً من تكرارها).
- التحقق من وجود صورة حقيقية لكل قطعة، وطباعة تحذير بالأكواد التي بلا صورة (دون إيقاف العملية).
- طباعة تقرير نهائي بعدد القطع المضافة والمحدَّثة.

### 4. ارفع التغييرات على GitHub Pages

بعد التأكد من التحديث، ارفع الملفات المعدَّلة (`data/products.json` وأي صور جديدة) إلى مستودع GitHub الخاص بالموقع:

```bash
git add data/products.json assets/images/products/
git commit -m "تحديث قطع الغيار"
git push
```

سيظهر التحديث على الموقع المباشر خلال دقائق قليلة من GitHub Pages.
